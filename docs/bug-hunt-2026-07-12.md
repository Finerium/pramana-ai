## 4. Ditangguhkan - butuh keputusan pemilik (stop-and-ask, aturan 1)

Menyentuh kontrak beku section 6 (tipe/skema/prompt/copy) atau perilaku desain
6.4/6.9; TIDAK diubah sepihak. Diurut dampak.

1. **PII pengurus/vendor dikirim tanpa masking ke LLM offshore** (`lib/audit/buildSnapshot.ts:110-133`, high). Aman utk data seed sintetis; untuk data koperasi NYATA, alamat rumah pengurus + identitas vendor (UU PDP) menyeberang ke prosesor pihak ketiga. Perbaikan (pseudonimisasi sebelum kirim) mengubah payload prompt 6.9. **Butuh keputusan adopsi: pseudonimisasi deterministik atau syarat DPA/region provider.**
2. **Prompt-injection via teks bebas bendahara** (`lib/audit/snapshot.ts:55`, high). `deskripsi`/`vendorNama`/`vendorAlamat` yang diketik pihak teraudit masuk mentah ke prompt forensik tanpa pemisah data/instruksi; injeksi "kembalikan {temuan:[]}" berpotensi memutihkan verdict. Model-dependent, dormant tanpa key. Perbaikan menambah framing anti-injeksi di aturan bersama 6.9 (beku).
3. **Adjudikator dapat mengosongkan/menurunkan temuan tanpa jejak** (`lib/audit/index.ts:273`, high). "warna server menang" tak melindungi dari pengosongan HIMPUNAN temuan. Mengubah perilaku 6.4 (mis. larang set-emptying, atau catat `temuanDrop`). Perilaku by-design -> keputusan produk.
4. **Kegagalan sebagian agen -> verdict berotoritas penuh** (`lib/audit/index.ts:245`, med). Brownout provider bisa membalik latestRun ke hijau dgn cakupan 1/4 agen; `agenGagal` tak dibaca surface mana pun. Ambang/threshold menyentuh 6.4.
5. **Periode = bulan transaksi terbaru menggeser audit ulang gov** (`buildSnapshot.ts:87`, med). Definisi 6.9 "500 baris periode berjalan (bulan tunggal)". Mitigasi non-kontrak: reset sebelum sesi re-audit gov.
6. **GovOverview kembalikan seluruh tabel koperasi tanpa paginasi** (`app/api/gov/overview/route.ts:35`, med). Aman di 12 seed; adopsi puluhan ribu unit butuh paginasi/agregat server = ubah bentuk respons 6.3.
7. **Deadline klien gov 120s < anggaran server** (`DetailClient.tsx:25`, med). "Periksa" rawan gagal-palsu di depan pejabat. Angka 120s tertulis blueprint 6.4.
8. **Beranda "Aktivitas Terbaru" statis dari copy** (`Beranda.tsx:475`, med). Di atas data nyata = fabrikasi. Angka ada di modul copy beku 6.15.
9. **Denylist vonis 6.5 tak memuat sinonim** (menipu/penggelapan/koruptor) (`lib/registerGuard.ts:21`, low). Daftar kata = kontrak beku 6.5.
10. **`--color-faint` gagal kontras AA** teks kecil (`styles/tokens/subjek.css`, low). Token diport verbatim design-handoff (nol downgrade 6.19).

**Item ke-11 (copy, gray-area):** perbaikan "Daftar: semua error jadi error NIK;
kegagalan lain (jaringan/500) tanpa feedback" (high) SUDAH siap tetapi
memerlukan **satu string copy baru** (`daftar.gagal`, register-compliant) karena
tak ada string existing yang cocok (`login.err` semantik keliru). Karena copy =
6.15 beku (aturan 1 = stop-and-ask), perbaikan ini **ditangguhkan** menunggu
persetujuan pemilik. Patch siap-terap: `scratchpad/deferred-bug2-daftar.patch`.
Logika (bedakan VALIDATION vs error lain) + banner sudah lengkap; tinggal restu
1 string.

## 5. Backlog medium/low (non-kontrak, terdokumentasi)

80 item medium/low tercatat lengkap di `scratchpad/triage.json` dan
`scratchpad/defer-brief.md` (skenario + usul fix per item). Tema utama:

- **Concurrency/observability:** worst-case rantai LLM bisa lewati anggaran 300s
  `after()` (marker orphan); semua kegagalan audit live ditelan tanpa log.

  jenis transaksi/pinjaman (temuan 'rasio'/'jadwal' lolos); retry korektif
  mengganti seluruh himpunan temuan (temuan bersih bisa hilang).

  LIVE di-mount saat konten muncul (tak diumumkan SR); sukses konsol tanpa
  role=status.

- **Performance hot-path:** endpoint pemeriksaan-berjalan full-scan json_extract
  tiap 4s; notif fan-out satu-statement per anggota.
